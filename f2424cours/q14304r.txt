В частности, есть инструменты, которые вы можете использовать вкак электронные таблицы, так и SQL для достижения аналогичных результатов.
Мы уже узнали онекоторые инструменты для очистки данных втаблицы, а это значит, что вы ужезнать некоторые инструменты, которые можно использовать в SQL.
Например, вы все еще можете выполнять арифметические действия,использовать формулы и объединять данные при использовании SQL, поэтому мыосновываться на навыках, которые мы изучили в электронных таблицахи использовать их для выполнения еще более сложной работы в SQL.

Слишком много данных для вводавручную даже для одной больницы.
Вот где SQL пригодится.
Вместо того, чтобы смотреть накаждый отдельный источник данных и записать его внашу электронную таблицу, мы можем использовать SQL для извлечениявся эта информация отразные места в нашей базе данных.

В электронной таблице мы можем использоватьфункцию СЧЁТЕСЛИ, чтобы узнать это, или мы можемобъединить запросы COUNT и WHERE вSQL, чтобы узнать, сколько строк соответствует нашим критериям поиска.
Это даст нам аналогичные результаты, но работает сгораздо больший и более сложный набор данных.

Во-первых, важно понимать, чтоэлектронные таблицы и SQL - разные вещи.
Электронные таблицы создаются с помощьюпрограмма, такая как Excel или Google Sheets.
Эти программы предназначены длявыполнять определенные встроенные функции.
SQL, с другой стороны,язык, который можно использовать для общенияс программами баз данных, такими какOracle MySQL или Microsoft SQL Server.
Различия междудва в основном в том, как они используются.
Если аналитик данных получил данныев виде электронной таблицы они будутвероятно, сделать их очистку данныхи анализ в этой электронной таблице, но если ониработа с большим набором данныхс более чем миллионом строкили несколько файлов в базе данных, это проще,быстрее и более воспроизводимо использовать SQL.
SQL может получить доступ и использоватьгораздо больше данных, потому что он может извлекать информацию изразличные источники в базе данных автоматически,в отличие от электронных таблиц, которыеиметь доступ к введенным вами данным.
Это также означает, что данные хранятся в нескольких местах.
Аналитик данных может использовать электронные таблицыхранится локально на их жестком диске илиперсональное облако, когда они работают в одиночку, ноесли они в большой команде с несколькими аналитиками, которыенеобходимо получить доступ и использовать данные, хранящиеся вбаза данных, SQL может быть более полезным инструментом.
Из-за этих различий электронные таблицыи SQL используются для разных целей.

SQL отлично подходит для работы с большими наборами данных,даже триллионы строк данных.
